# Security Policy for NeoTutor-AI

## Supported Versions  
Security updates and maintenance are provided only for the latest version (main branch) of NeoTutor-AI. Older versions and forks may not receive security fixes.

## Security Best Practices for Contributors  
When contributing to NeoTutor-AI, please follow these guidelines:  
- **Do not commit any credentials**, API keys, tokens, or secrets.  
- **Avoid adding dependencies** without reviewing their source, license, and security history.  
- **Avoid using unverified external scripts or CDN resources** that could introduce vulnerabilities.  
- Run appropriate security audits (e.g., `npm audit`, `yarn audit`, or equivalent) before pushing updates.  
- Review proposed changes for security implications (e.g., data exposure, injection risks, permissions escalation).  
- Report any suspicious commits, pull requests, or unexpected behavior immediately.

## Responsible Disclosure  
We thank you for responsible disclosure of issues. We kindly ask that you **refrain from publicly disclosing** any security vulnerability until:  
1. The issue has been verified and a fix has been released (or scheduled).  
2. A public advisory is published by the maintainers (if needed).  

Valid reports will be credited in release notes (unless anonymity is requested).
